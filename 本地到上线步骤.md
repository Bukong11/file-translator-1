# 真实翻译网页上线步骤

这个文件夹 `file-translator-deploy` 是可以上传 GitHub 的干净版本。

## 你要上传 GitHub 的内容

上传整个文件夹里的内容：

```text
backend
frontend
render.yaml
.gitignore
README-DEPLOY.md
```

不要上传：

```text
.venv
start-backend-with-openai.bat
测试生成的 docx/pdf 文件
```

## 第一步：部署后端 Render

1. 打开 Render
2. 用 GitHub 登录
3. 选择 `New +`
4. 选择 `Blueprint`
5. 选择你的 GitHub 仓库
6. Render 会读取 `render.yaml`
7. 在环境变量里填写 `OPENAI_API_KEY`
8. 部署

部署完成后，你会得到一个后端网址，例如：

```text
https://file-translator-backend.onrender.com
```

测试：

```text
https://file-translator-backend.onrender.com/health
```

看到 `translation_mode` 是 `openai` 就说明后端成功。

## 第二步：连接前端

打开：

```text
frontend/config.js
```

把里面的：

```js
window.FILE_TRANSLATOR_API_BASE_URL = "https://your-render-backend-url.onrender.com";
```

改成你的 Render 后端网址。

然后提交到 GitHub。

## 第三步：部署前端 Vercel

1. 打开 Vercel
2. 用 GitHub 登录
3. Add New Project
4. 选择同一个仓库
5. Root Directory 填：

```text
frontend
```

6. Deploy

部署完成后，别人打开 Vercel 网址就可以上传文件真实翻译。

## 注意

API Key 只能放 Render 环境变量，不能放网页代码里。

PDF 专业版可以尽量保留图纸版式，但扫描图纸还需要 OCR。
